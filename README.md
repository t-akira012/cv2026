### 2019年11月〜2021年3月：新聞社データ配信プロジェクト（インフラチームリーダー）

**概要**

大手通信社のスポーツニュース配信システムにおけるAWSインフラ基盤開発。IBM製オンプレミス業務システムのWebリプレイス案件。

**プロダクト概要**
| 種別 | 内容 |
|------|------|
| 社内 | スポーツニュース配信システム（本番/ステージング/災害対応） |


**技術スタック**

| 領域 | 技術 |
|------|------|
| インフラ | AWS / Docker |
| バックエンド | Python / Django |

**システム要件**
- ほぼ同一コードベースの3サービスを構築する必要があり、加えて通信社の特殊要件として、AWS環境（本番/ステージング）、オンプレミス（災対環境）、ローカル開発環境の3環境での動作が求められた。
- ステージング環境については廃棄容易性の確保も要件に含まれていた。

**担当業務**
- 参画時点ではAWS・Docker専任担当者が不在であり、DevOpsに適さないコードベースの改善から着手しました。
- 具体的には、ハードコーディングされた環境変数の外部化や、AWS ECSにおけるawsvpc/defaultモード差分を吸収するためのコンテナ間通信の修正を実施。
- Heroku 12 Factor Appに基づくコンテナ設計方針をチーム内に周知しながら、並行してAWSクラウド基盤を構築した。

**利用AWSサービス**

RDS / VPC Endpoint / SQS / S3 / CodeBuild / ECS Fargate / ECR / ALB / ACM / WAF / AWS Shield Standard / EFS

**技術的な工夫**

- Heroku 12 Factor Appに基づくコンテナ設計方針の策定・チーム内周知
- 環境変数のハードコーディング外部化、コンテナ間通信の環境差分吸収（docker-entrypoint.sh等）
- CloudFormation化後、AWS CLIによるShell Scriptで6環境（3サービス×本番/ステージング）への複製を自動化
- ALBエンドポイント固定によるDNS影響の回避
- スケジュール制約下での優先度判断（一部はコンソール構築＋手順書で再現性担保）

**発揮したバリュー**

- 参画前はAWS・Docker非専任体制でDevOpsに適さない状態だったコードベースを、移植性・再現性の高い基盤へ改善。
- チームへのベストプラクティス周知と並行してインフラ構築を推進。

### 2019年〜2020年：自治体災対MAPアプリ開発（インフラエンジニア / Webディレクター）

**概要**

某自治体の災害対策Mapアプリをゼロから開発。AWS ECSをベースとしたモダンなアプリケーション構築。

**技術スタック**

| 領域 | 技術 |
|------|------|
| インフラ | AWS / Docker / ECS |
| フロントエンド | React / TypeScript / Webpack |
| 負荷テスト | JMeter |

**担当業務**

| 役割 | 内容 |
|------|------|
| インフラエンジニア | Dockerコンテナ設計、AWS ECS構成設計、負荷テスト、Node.js環境構築 |
| Webディレクター | アジャイル開発フロー構築、工程管理（フロント/テスト/デザイナー）、画面設計仕様書・テスト仕様書作成、契約締結〜納品 |

**対応内容**

開発体制が存在しない状態からの立ち上げとして、以下の順序で対応。

1. **アジャイル開発体制構築**
   - Slack / Scrapbox / GitHubを活用したコミュニケーション・ドキュメント共有フローを設計
   - 用途別チャンネル構成（main / dev / design / todo / github notify）によるチーム連携体制を確立

2. **画面設計書作成**
   - 自治体案件の保守性要件を踏まえ、他自治体の類似サービスを参考に設計

3. **Docker設計・ECSデプロイ対応**
   - 他社AWS環境への納品要件によりCI/CDパイプライン構築不可
   - npm scriptでビルドしたReactAppをDocker build可能な状態で生成するShell Scriptを作成
   - 社内AWS環境で動作確認後、Dockerfileを納品

4. **テストフロー構築**
   - GitHub Projectsを活用したKanban方式でフロントエンジニア→テストチーム間の高速テスト体制を確立

5. **負荷テスト実施**
   - JMeterによるシナリオ作成・実行

**発揮したバリュー**

インフラとマネジメント両面の知見を活かし、開発体制ゼロの状態からアジャイル開発を成功に導いた。会社間制約でCI/CD構築は叶わなかったものの、React + Webpack + TypeScriptアプリケーションのnpm script導入からShell Script連携によるDockerコンテナ化、ECS動作までを実現しました。
